A) Introduction

Urban Crosser is a 3D HTML5 game built with Three.js. It challenges players to cross busy city lanes filled with traffic and obstacles while collecting points. Featuring realistic lighting, moving vehicles, animated characters, and touch/keyboard controls, Urban Crosser delivers an immersive endless runner experience directly in the browser.

B) Requirements

  • Browser: Chrome, Firefox, Edge, or Safari (latest versions)
  • Optional: Local web server (for smooth performance)
  • Basic knowledge of HTML/CSS/JavaScript for customization

C) Installation & Setup

  1. Unzip the downloaded package.
  2. Open index.html in a modern web browser.
  3. To edit or customize, open files in your code editor (VS Code, Sublime, etc.).
  4. For deployment, upload all files to your hosting server inside the public_html/ folder.

D) Folder Structure

UrbanCrosser/
├─ index.html        # Main HTML file (contains structure and embedded styles)

└─ assets/
   ├─ textures/      # Optional folder for background/road textures
   └─ sounds/        # Sound effects (if added)

E) How the Game Works

Urban Crosser generates a series of random lanes containing grass, forests, and roads with moving vehicles. The player controls a character who must cross the city while avoiding obstacles.

  • Objective: Move forward across as many lanes as possible without getting hit.
  • Environment: Dynamic city roads, forests, and vehicles with lighting and shadows.
  • Engine: Built on Three.js for 3D rendering with optimized WebGL performance.
  • Game Flow: Start → Play → Score → Game Over → Retry.

Customization Tips

  • Modify lane counts, vehicle speeds, and light intensity to adjust difficulty.
  • Change character color or geometry inside the createCharacter() function.
  • Update background gradient or skybox for different visual themes (day/night mode).

H) Troubleshooting

  • Game not loading? Make sure Three.js CDN link is active in index.html.
  • Lag or low FPS? Try running from a local server (like Live Server in VS Code).
  • Controls unresponsive? Ensure your browser window is focused and active.

I) FAQ

Can I add sound effects? Yes, integrate the HTML5 Audio() API or import small sound files into the assets/sounds folder.

Is it mobile-friendly? Yes, the game supports both touch (swipe) and on-screen buttons for mobile users.

Can I change lighting or theme? Absolutely. Adjust the DirectionalLight and HemisphereLight properties in the script.
